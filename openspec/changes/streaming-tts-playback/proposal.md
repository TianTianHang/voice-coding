## Why

当前后端只有 STT 和录音链路，没有语音输出能力。为了支持“先合成、后播放”的语音助手体验，需要新增 TTS 后端能力，并且在播放期间暂停录音，避免扬声器回灌影响 VAD 和转写流程。

## What Changes

- 新增后端 TTS 能力的抽象层，采用与 STT 类似的 trait + 实现层结构。
- 新增 TTS 合成流程，将文本合成为可播放的音频数据，目标输出格式为 48kHz、立体声。
- 新增后端播放流程，在合成完成后再播放音频，而不是边合成边直接输出到扬声器。
- 播放期间暂停录音/VAD 会话，播放结束后恢复监听。
- 新增与 TTS 生命周期相关的运行时状态、错误处理和 Tauri 命令。

## Capabilities

### New Capabilities
- `streaming-tts-playback`: 后端 TTS 合成与播放能力，包含文本合成、音频播放、播放期间暂停录音以及运行时状态管理。

### Modified Capabilities

- `backend-audio-recording`: 播放期间需要与录音会话协作，保证录音可被暂停和恢复。
- `backend-vad`: 播放期间 VAD 需要进入暂停/空闲语义，避免把扬声器输出误判为用户语音。

## Impact

- 后端将新增 TTS 相关 Rust crate / 模块、运行时状态和 Tauri 命令。
- 音频层将新增输出播放逻辑，和现有输入录音逻辑并列。
- 需要更新后端会话控制，确保播放期间不采集麦克风音频。
- 需要新增 Rust 测试覆盖状态切换、音频格式约束、播放暂停与恢复流程。
