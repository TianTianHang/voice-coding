┌─ ⚠️  重要规则 ────────────────────────────────────────────────────────┐
│                                                                          │
│  1. 必须使用 --test-threads=1                                          │
│     （否则会因为内存不足被OOM killer杀掉）                              │
│                                                                          │
│  2. 确保模型文件存在                                                    │
│     models/ 目录包含2.5GB的ONNX模型文件                                 │
│                                                                          │
│  3. 设置 ORT_DYLIB_PATH                                                │
│     export ORT_DYLIB_PATH=/path/to/libonnxruntime.so                    │
│                                                                          │
│  4. 最小音频要求                                                        │
│     边界测试中音频至少2秒（32000 samples @ 16kHz）                      │
│                                                                          │
└──────────────────────────────────────────────────────────────────────────┘

┌─ 手动运行命令 ────────────────────────────────────────────────────────┐
│                                                                          │
│  # 只运行单元测试（快速）                                                │
│  cargo test --lib                                                       │

┌─ 故障排除 ────────────────────────────────────────────────────────────┐
│                                                                          │
│  问题: SIGKILL (signal: 9)                                              │
│  原因: 内存不足（OOM）                                                  │
│  解决: 添加 --test-threads=1                                            │
│                                                                          │
│  问题: assertion failed: result.is_ok()                                │
│  原因: 音频时长不足                                                     │
│  解决: 使用至少2秒音频（32000 samples）                                 │
│                                                                          │
│  问题: Failed to initialize STT engine                                  │
│  原因: 模型文件缺失                                                     │
│  解决: 确保 models/ 目录包含所有必需文件                                │
│                                                                          │
